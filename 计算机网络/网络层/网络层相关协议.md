**IP地址划分**

网络ID就是各个划分的局域网之间的地址，路由器按照网络ID转发分组

**IP地址类别**


![[Pasted image 20250331122016.png]]
ABC类网络地址的区别是网络ID和主机ID所占位数不同


**怎么区分不同类别的地址，由开头几位来区分**
![[Pasted image 20250331122146.png]]

**原理：**


![[Pasted image 20250331123100.png]]

对于8位二进制数进行如图上的划分，就是对其进行二分操作，将0-128划分为A类，128-192划分为B类，而192-224划分为C类以此类推进行不同段的划分，所以我们只看IP地址的前8位就能判断出是哪类地址

![[Pasted image 20250331123803.png]]

这里有个小插曲，对于A类地址的第127号，被认为代指主机本身，也就是访问该地址相当于访问自己的主机，IP地址为127.0.0.1




**子网掩码的作用**：

![[Pasted image 20250331124242.png]]

子网掩码就是告诉计算机IP地址中那部分是网络部分那部分是主机部分，这个可以用来判断两台主机是否同属于一个子网，对应的网络部分都是255,这样用子网掩码和IP地址进行按位与运算得到的结果就是网络ID，主机部分归0，就可以对不同的IP地址进行比较了

![[Pasted image 20250331124534.png]]




### 子网划分

![[Pasted image 20250331131122.png]]

以C类地址为例，如果想将原本处于同一个子网的网段继续划分为两个网段，那么可以将最后八位的范围二分这样后八位就分为了1-126,129-254两个范围（PS：主机IP不能为全1和全0）

子网掩码的后八位也进行改变，将最高位改为和IP地址后八位的最高位相同就能用子网掩码相与进行判断，**相与之后的值就是网络ID**


**划分为四个子网**：

![[Pasted image 20250331194302.png]]

同理等分为8个子网也是相同的


**之所以要进行子网划分是因为要最大限度的合理分配有限的IP地址**，比如如果一个只有两台主机的子网，怎么进行划分？那么就是二分六次，从而主机ID只有1,2两个可取

上面的例子是针对于C类网络进行的划分，其他两类网络原理类似


#### **变长的子网划分**：

![[Pasted image 20250331200049.png]]

**对于等长划分来说**，我们划分的方式是将原IP的剩余主机号重新进行划分

![[Pasted image 20250331222114.png]]

例如对于这些网络地址，“/”  后的数字表示网络ID的位数，后面的数字则是具体的主机ID，我们如果继续在这些网段上进行划分子网，则是针对于原主机ID继续划分，比如对于等长，如果我们需要划分N个子网，则占用的最少位数为 $log2(N)$,

而对于不等长划分来说我们按照实际需要进行划分，例如上图红笔标记的需要分别划分100,50,20,10台主机，对于100来说可以放在128-255段内，而50可以放在64-127内以此类推，划分一百台主机可能占用1位，而划分50台主机就占用2位，我们只需要改变这两个的子网掩码就可以，比如100台主机的子网掩码就是255.255.255.128，而50台主机的子网掩码就是255.255.255.192，仍然可以确保子网和子网掩码运算和得到的网段唯一


**最后需要注意的是，主机ID不能全0或者全1，全1为子网的广播地址**




#### 超网

对于同一个网段的两个子网，如下图所示
![[Pasted image 20250401104832.png]]

这两个分属两个子网本来是不能通信的，如果想要通信需要路由器进行转发，但是他们的交换机直接相连，如果还通过路由器进行转发那会降低效率

**合并网络**：如果将两个子网合并那它们之间就就能通讯了

![[Pasted image 20250401105216.png]]

合并的方式就是将子网掩码向前移动一位并且后续的位数为0，使得该子网掩码和两个IP与运算得到的结果相同，注意这里只修改了子网掩码而没有修改其原网络ID，如果网络ID的后缀是00,01,10,11那么子网掩码应该向前移动两位进行合并

![[Pasted image 20250401110323.png]]

需要注意的是不是任意两个单独的网段都能合并，例如01，10就不能单独合并，如果要合并这两个网段那么00和11这两个网段也被合并了，判断的方法，如果 网段%4，最后是00,01或者是10,11就可以单独合并



### IP地址和MAC地址


在端到端通信的过程中，数据帧可能要进行多个存储转发过程，过程中数据帧的源IP和目标IP是不变的，而MAC地址每通过路由或者计算机转发一次都要改变，**MAC地址决定了下一跳去哪，而IP地址决定了最终去哪**

由此有一个推论就是，本网段的计算机只能发现本网段计算机的MAC地址，如果另一个网段的MAC地址经过路由器转发后源MAC地址变为了路由器的MAC地址，就发现不了了，所以夸网段的计算机通信只能通过IP地址来管理
![[Pasted image 20250401111408.png]]

为什么不只用MAC地址，因为全球的计算机不可能都只通过交换机相连，况且如果只通过交换机相连无法区分网段，没有安全性



#### IP数据报（数据包）
![[Pasted image 20250401144413.png]]

关键字段：
首部长度：一个数据报首部的长度，占4位所以首部最长是60字节
区分服务：区分该数据报被转发的优先程度
总长度：数据报的长度，最大为$2^{16}-1$字节，总长度不超过MTU，MTU就是能够传输的最大数据包长度，取决于传输的网络
标识：
![[Pasted image 20250401145647.png]]

标志和偏移量：
标志标识有没有被分片和是否后面还有分片

什么是分片？
当主机发送的数据包比较大，而中间路由器无法处理这么大的数据包时就将数据包进行分片，如下图：
![[Pasted image 20250401151659.png]]
分片之后在每个分片之前在加上首部，偏移量指的就是每个分片的开始距离起始数据的位移计算为分片起始字节/8，即偏移量以八个字节为一个单位计算


生存时间（TTL）：8位，数据包在网络中可通过的路由器的数的最大值，每次经过路由器TTL的值就会减一，当路由器识别到这个包的TTL的值为0时就不再转发
这是为了防止数据包在网络中循环占用网络资源

协议：标识数据是什么东西

首部校验和：用来校验首部在传输过程中是否出错





### 数据包转发流程


![[Pasted image 20250401161212.png]]

如图，0计算机的数据包想要到1计算机，那么就要求中间所有的路由器都知道数据包该怎么走
![[Pasted image 20250401161325.png]]
路由器内有维护有路由表，路由表中记录了目标地址和下一跳之间的关系，也就是沿途每个路由器都知道要到目标地址下一步该怎么走，路由表是提前配置好的，这叫做**静态路由**

### 动态路由协议


**RIP协议**：
将自己知道网段周期性广播给其他路由器

![[Pasted image 20250401170109.png]]
比如A知道去1网段该怎么走，A就广播给B和F，B知道A能够连接1网段，B也更新路由表如果目标地址是1网段则下一跳给A，并且记下去1网段中间需要经过多少个路由器，B也进行广播，以此类推

对于E路由器来说，它的路由表里有两条1网段的路径，它优先选择经过的中间路由器少的那条

可以看出RIP协议能够让网络中的路由器自动更新路由表减少了手动更新

# ARP协议


![[Pasted image 20250401113911.png]]

ARP协议的作用就是将IP地址转为实际的物理地址，比如通过交换机相连的两台计算机它们之间是只能通过MAC地址确定对方的，因为交换机只能识别MAC地址，但是开始的时候互相并不知道对方的MAC地址只有IP地址，这时候需要进行广播，通过广播的应答和ARP协议将IP地址转换为MAC地址，这样对方就能知道MAC地址了

# VPN（虚拟专用网络）技术

![[Pasted image 20250401172640.png]]


如果我们主机想要访问一个私网地址，那么正常情况下是无法访问的，因为路由器没有这个路径，我们在私网和公共互联网之间架设一台服务器，服务器一边连私网一边连互联网，我们直接访问RAS这个服务器，服务器会分配给我们一个私网地址，我们就可以通过它来访问私网，我们送给RAS的数据包经过的两层包装，内部是实际访问私网的数据包目标地址和源地址都是私网地址，外部又加了一个首部是公网地址，这样数据包就能通过公网访问私网

# NAT  （网络地址转换）


 当私网的主机想要访问公网的主机时，数据报能从私网到公网但是，无法从公网相应会私网，因为公网路由器没有私网主机IP的路由
 解决方法就是数据包发送经过私网的网关时将数据包源地址改为当前网关分配的公网地址，这样公网路由器就能找到该地址，再通过该地址将数据包转发给私网路由器

网络地址转换 NAT（Network Address Translation）就是用来解决本地编址的内部网络与外网通信的问题：

- 需要在内网与外网的边缘路由器上安装 NAT 软件，装有 NAT 软件的路由器叫做 NAT 路由器，它至少有一个有效的外部全球地址 IP
- 所有使用本地地址的主机在和外界通信时都要在 NAT 路由器上将其本地地址转换成全球地址 IP 才能和因特网连接

#  VPN 与 NAT 的区别

可以发现 VPN 协议和 NAT 协议都是通过重新构建一个 IP 首部来实现的，但他们的实现又有区别，VPN 是将内部 IP 数据报加密后打包成外部 IP 数据报的数据部分，它的主要目的是为了数据的保密性，而 NAT 是纯进行地址转换，它的目的是为了解决本地编址的内部网络与外网通信的问题。
