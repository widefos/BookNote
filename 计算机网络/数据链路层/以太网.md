**网络适配器介绍**：
网络适配器属于计算中的硬件，有RAM和ROM,适配器和局域网之间的通讯是以串行的方式通讯的，而在计算机内部数据则是并行的，所以适配器的一个重要作用在串行和并行数据之间转换
适配器还需要操作系统有对应的驱动软件，并实现相关以太网协议



**多点接入：** 表示计算机以多点连接的方式接入总线

**载波监听**： 每个站在发送数据之前先检测一下总线上是否有其他计算机在发送数据如果有则暂时不发送数据
**PS**: （这里所说的站指的是局域网中的计算机，**主机**，**工作站**等名词可认为是同义词）



**碰撞检测**：碰撞检测是载波监听的实现方式

![[Pasted image 20250328212732.png]]

**为什么会碰撞**？

![[Pasted image 20250328213202.png]]

$A \to B$ 发数据，在 $t=\tau -\delta$ 时，数据发生碰撞，也就是$B \to A$ 发的数据和$A \to B$
的数据相遇了，此时两方的数据叠加之后继续传播，而双发要检测到数据发生碰撞需要等到叠加的数据被对方接受到之后才能检测到，对于$A$ 来说从发出数据开始到检测到碰撞的最大时间为$2\tau$, $\tau$为A到B数据传输的延迟

根据这点可以得出： **如果发送数据后 $t>2\tau$,那就说明没有碰撞产生，从发出信号到$2\tau$这段时间内叫做以太网的争用期，只有经过这段时间才能确定发出的数据没有碰撞

![[Pasted image 20250330144444.png]]
这里的最短有效帧长是针对于**接收方**来说的，接收方如果收到的帧小于64字节那就说明中间发生了碰撞就丢掉该帧


因为双方同时通信会产生碰撞，所以该协议只能进行**半双工通信**


**发生碰撞后怎么办？**
![[Pasted image 20250330150016.png]]

基本思想就是检测到碰撞之后，等待一段时间之后在此发送，如果两端计算机随机到的$r$的值相同那么下次重传还是会发生碰撞此时 $k$ 就变多，$r$ 的可取范围也就扩大了，取到同一个数的概率就会变小了


![[Pasted image 20250330153037.png]]

这里由于现阶段很多只有MAC协议所以我们只讨论MAC协议



### 以太网信道利用率

以太网信道上的情况，争用期和占用期
![[Pasted image 20250330155355.png]]

**利用率参数a**：

$$
a = \dfrac{\tau}{T_0}
$$
$\tau$就是信道端到端的发送延迟，而$T_0$ 则是占用期的时间，这个公式代表着，端到端之间的线路越短，争用期的时间越短，a就越大，a所反应的就是争用期和占用期所占之间比例的情况

极限利用率：
$$
S_{max} = \dfrac{T_0}{T_0 + \tau} =  \dfrac{1}{1+a}
$$
极限利用率指的是发送过程中没有碰撞的情况也可以说是以太网的最大利用率

### 集线器


![[Pasted image 20250330153746.png]]

集线器只负责将所有接口的计算机逻辑上接入同一个总线，不做其他操作，所以它是工作在物理层的，也不转发数据，所以在数据链路层上集线器相当于是透明的




### 以太网的扩展

**用集线器的扩展**

![[Pasted image 20250330164448.png]]

如果这样扩展，那么由于总线性结构的缺点，两台计算机通信时信道被占用其他计算机无法通讯，冲突的概率变大了，所以如果都用集线器来进行连接那么网络整体效率会降低


**网桥**

![[Pasted image 20250330170248.png]]

网桥怎么转发处理数据？

图上的网桥有两个接口连着两个局域网，比如A和B通讯，A的数据要经过网桥
这时候网桥会查询内部维护的一张表，这张表记录了主机和接口之间的关系，那个主机对应哪个接口，如果没有记录A那么就记录下发送端的地址和对应接口，并将该数据转发到所有接口，如果目标地址能够在表中查到那么就转发到对应接口

PS:为什么只记录源地址而不是目标地址？
因为网桥只知道经过它的数据从哪个接口来，如果表中没有目标地址那么它就不知道这个地址对应哪个接口，所以只能记录源地址的接口对应关系，如果没有目标地址接口则向所有接口转发

![[Pasted image 20250330170930.png]]


**当网桥上的接口足够多就可以每个接口连一台单独的主机时，而不用用集线器，那实际上就是交换机**

**为什么能够提高效率？**

当两台计算机之间在通讯，这时另一台计算机也想通讯通讯时，交换机内部可以让需要交换的数据进行排队转发，所以就没有冲突了，利用率提高了

#### 虚拟局域网（VLAN）

**什么是LAN**?
LAN就是一个接在路由器上的一个局域网


**什么又是VLAN？**

真实需求中，我们要求有些不在同一局域网的计算机同时又能够共享资源，也就是我们么要求这些计算机在逻辑上构成了一个局域网不论其实际物理连接如何，这样划分出来的逻辑局域网就是VLAN

这样的好处在于，同一个逻辑局域网内的主机间可以共享资源而处于不同VLAN之间的主机无法直接通讯

![[Pasted image 20250330212105.png]]

![[Pasted image 20250330211712.png]]

**VLAN如何夸物理连接？**

![[Pasted image 20250330214648.png]]

这里需要区分**干道链路和访问链路**，如果可以承载多个VLAN的数据的链路就是干道链路，而只能承载一个VLAN的链路就是访问链路，通过干道链路访问时需要在帧中加入VLAN相关的标识使得对应的交换机能够识别那些端口属于同一个VLAN
