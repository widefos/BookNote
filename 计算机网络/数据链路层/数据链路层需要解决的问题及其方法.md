# **三个基本问题**：

* 封装成帧
* 透明传输
* 差错控制

### 封装成帧

数据链路层接收网络层传输的IP数据包，并将报文拆分为数据帧

在一段数据的前后分别添加首部和尾部就构成了帧，首部和尾部的作用是确定帧的界限

当发送端发送一个帧时，如果发送中断或者出现问题需要重新发送，而接收端此时已经收到了一部分数据时，这时接收端并没有读到数据的尾部或者帧结束符，所以此时接收端应该放弃这段数据，重新接收


### 透明传输

解释下**透明**这个词的意思，在这里透明指的是无论传输什么样比特组合的数据，数据链路层都不会阻碍数据的传输，数据可以完整的通过它而不会出现差错，也就是说数据链路层对于数据来说好像是透明的一样


**为什么需要这个概念？**

![[Pasted image 20250327195632.png]]

如果传输的数据为可打印字符集也就是ASCII中的内容组成时不会出现冲突（因为ASCII可打印字符都能从键盘直接输入而帧控制符不会使用这些编码所以不会有冲突）这样的传输是透明的，而如果不是仅仅由“ 可打印字符”组成时，就会出现问题，数据当中可能会出现帧开始符和帧结束符，导致部分数据被当做无效帧舍弃，这样数据链路层就会阻碍数据的传输

#### 解决方法

**字节填充法**：

当发送端的数据当中出现帧控制字符时在其前面加入转义字符**ESC**，（**这里数据的单位是字节**，接收端也按字节读取），这样接收端读到这里时如果前面有一个**ESC**就知道这是数据内容的部分而不是控制符

### 差错控制

传输过程中可能会受到各种干扰导致传输的比特产生差错例如1变为0或者0变为1

一段时间内，传输错误的比特和传输数据的总位数的比值称为**误码率**，也就是传输错误的内容在总的传输数据中占了多少


#### 循环冗余检验(CRC)

**冗余码概念:** 在原本要传输的数据后面加上的用语检测数据传输是否有错的多余位数

循环冗余检验是一种检错码，用来检测数据传输过程中是否有错

**基本原理:**


模2除法运算

![[Pasted image 20250327204900.png]]
这里需要注意的是模2除法和普通除法的不同，模2除法只要某一位的商和除数相乘所得到的最高位和被除数最高位相同就是够减的，例如这里最开始商1得到1110，和被除数的前四位比较最高位都为1说明够减，然后对这四位进行异或运算得到结果，反复如此，最终得到商和余数


而我们用模2除法来规定了这样一种检验方法：**约定除数，检验余数**，
也就是如果最后接收端能够用约定书除数除尽就说明该数据没有错误，如果存在余数则说明传输过程中存在差错

![[Pasted image 20250327205433.png]]

这里为什么在原数据后添加的冗余码是四位？

因为这里原数据是八位数据，约定的除数为五位数，余数最多为4位，添加4位可以保证这里检验的时候如果数据是正确的能够被除尽

**如何得到冗余码？**

1. 首先先看我们约定的除数位数，冗余码位数 = 约定除数位数-1
2. 将原数据补足冗余码位数都为0
3. 将得到的数和约定的除数进行模2除法
4. **得到的余数就是冗余码**

![[Pasted image 20250327210525.png]]

这里的结论比较重要，也就是如果约定了除数和给出了原数据，那么最后所得到的冗余码一定是唯一的
**结论应用：如果接收方收到了数据那么就用约定除数除以该数据看看余数是不是0，或者计算冗余码是不是接收到的数据相同即可**

这就是**CRC**，这里还需要注意**CRC**是检错码而不是纠错码，只能检测传输数据是否存在错误，如果要做到可靠的传输还需要加上**重传**和**确认**机制

# PPP协议(Point-to-Point Protocol)

世界上使用最多的数据链路层协议

使用场景，如果用户使用拨号接入互联网时，使用的是PPP协议，可以记录用户的身份信息进行身份验证

PPP协议需要满足的要求：

* 简单
* 封装成帧
* 透明性
* 多种网络层协议 （这表示该协议可以支持多种上层协议）
* 多种类型链路
* 差错检测
* 检测连接状态
* 最大传送单元
* 网络层地址协商 （拨号成功后分配IP地址）
* 数据压缩协商
