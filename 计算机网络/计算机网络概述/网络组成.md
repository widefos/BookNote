

# 主机通讯方式

**客户/服务器方式（C/S）**: 客户端向服务器发出请求，服务器响应客户端提供服务，比如请求网络页面就是C/S方式，当单个服务器请求过多时会造成负载压力

**对等方式（P2P）**:主机即可以充当客户端也可以充当服务器，例如P2P下载，如果有多个用户都下载了同一文件，那么主机就可以从多个源进行下载

# 网络数据交换

**电路交换**：
![[Pasted image 20250326093058.png]]

通讯设备通过交换机相互连接，需要通讯时由交换机进行转接
![[Pasted image 20250326093503.png]]
比如打电话的过程需要在两个用户之间建立连接，拨号就是在建立连接商申请资源，而挂断电话就是释放连接资源的过程

应用场景：适合于**数据传输量大**和**稳定性要求高**的**实时场景**，例如语音和视频通话就是建立了一条专线连接通路，此外核心路由器之间也可以使用电路交换

---

**分组交换**：
![[Pasted image 20250326094543.png]]
对于发送端来说，把数据拆分为多个分组，每个分组的首部携带有源地址和目标地址等关键信息，将每个分组独立发送出去

接收端收到数据之后，去除首部然后将各个分组数据拼接为源数据

![[Pasted image 20250326094852.png]]
由于每个分组是**独立进行发送**的，所以分组转发的过程每个分组到接收端所走的中间路程可能不同，**可能会有多个路径**

==*路由器功能*==：存储转发，存储可以存储数据包走过的路径，路由器内部接收到分组数据包后会查找转发表转发给下一个

优点：高效，灵活，迅速，可靠
问题：时延，开销

---

**报文交换**：和分组交换类似，也是使用路由器进行转发，但是报文交换每次传输的单位是报文，**不进行额外分组**，也会在数据前加上首部包含源地址和目标地址等信息

报文交换的延迟比分组交换要长
