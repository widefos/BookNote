## 使用痛点

| 问题 | 说明 |
|------|------|
| 🔴 账号封禁风险 | 活跃 IP 区易被封，Anthropic 对国内用户/IP 不稳定敏感，充值后也可能被封 |
| 🌐 网络访问困难 | 需要稳定可用的科学上网工具 |
| 🖥️ CLI 不友好 | 命令行界面非程序员上手难度大 |
| 🔊 BGM 过大 | 视频背景音乐音量太大，影响学习 |

---

## 核心技巧

### 常用命令

| 命令          | 作用                          |
| ----------- | --------------------------- |
| `cc switch` | 管理和切换模型                     |
| `/init`     | 项目初始化，推荐用 DeepSeek V4（价格优势） |
| `/context`  | 查看上下文，60% 任务**手动压缩**效果更好    |
| `/resume`   | 恢复对话，回到之前状态                 |

### 其他技巧

- **版本管理**：用 Git 配合回滚，比 CC 自带回滚更稳定
- **中文显示**：`Ctrl+Shift+Y` 打开搜索栏，搜索 `chinese` 可切换中文界面
- **提高准确率**：先生成代码再测试，通过迭代优化
- **插件推荐**：使用 **Superpower** 插件增强 CC 功能

## 主流 AI 编程工具对比

### 高频替代品

| 工具                  | 特点                                |
| ------------------- | --------------------------------- |
| **OpenAI Codex**    | 最强且用户友好的替代品，适合非程序员                |
| **OpenCode**        | 开源，支持第三方 API，不易被封                 |
| **Cursor**          | 图形化界面，用户友好                        |
| **Gemini**          | 性能强，价格有优势                         |
| **DeepSeek**        | 成本低，作为 API 接入 CC 时性能优秀，但复杂项目可能不稳定 |
| **Trae / Windsurf** | 可作为辅助工具                           |

### 混合使用策略

> Claude Code 负责规划、思考、检查
> Codex 或 DeepSeek 负责执行具体任务

——在质量和成本之间取得平衡。

---

## 模型推荐

- **首选 DeepSeek**：成本低，稳定性好
- **CC Switch**：可登录和切换不同 API，操作简便

---

## 成本与定位

- **目标用户**：有编程基础的专业开发者
- **成本**：Claude API 价格较高，DeepSeek 等国产品牌提供低成本替代
- **市场现状**：美国市场 CC 用户多为对价格不敏感的高收入群体（年薪 10 万美元以上）
