自上而下代表从语法树的更节点进行推导，直到叶节点

从文法上来说就是从开始符号进行推导，直到推导出整个句子

## 递归下降分析器

进行语法分析，需要对应的对于文法规则的分析方法，其中比较通用的是递归下降分析器


![[Pasted image 20250413211349.png]]


我们以这个文法来说，假设我们现在有串  $x*y$  

递归下降分析器先读入x然后从S开始递归分析文法进行匹配，第一个匹配的字符是x，所以现在匹配第二个字符 *，A进行推导，由于A的两个产生式开头都是 *  无法区分，所以需要两个都尝试，那么由于这个是两个字符指针需要向前移动继续读入字符，读入的字符是y那么说明这个匹配错误需要倒退回A进行重新分析选择产生式

这就产生了**回溯**，回溯就是当前判断错误需要倒退回之前的状态重新选择

## 左递归问题

![[Pasted image 20250412163910.png]]

这就是递归下降分析的弊端所在，需要回溯意味着面对复杂文法，时间复杂度可能会很高，而自上而下的分析过程中则不允许产生左递归，计算机是没有办法处理左递归问题的，因为会无限制的推导下去。所以需要消除左递归

所以自上而下分析主要解决两个问题

* 消除左递归
* 解决回溯问题



计算机是没有办法处理左递归问题的，因为会无限制的推导下去。所以需要消除左递归





### 直接左递归消除
![[Pasted image 20250412163941.png]]

左递归一直推导相当于产生递归的非终结符后的符号串一直重复下去，我们把这串在递归中重复生成的符号串重新起个名字叫做A'，如图所示，然后将A‘转换为右递归的形式并且可以以空串结束，这样就消除了左递归

**核心思想是将左递归转换为右递归，引入新的非终结符。**



### 间接左递归消除


**原始文法**：
    $$𝐴→𝐵𝑎∣𝑐$$$$𝐵→𝐴𝑏∣𝑑$$​
- **排序非终结符**：假设 A 在 B 之前。
    
- **替换 B 中的 A**：
    
    $$𝐵→(𝐵𝑎∣𝑐) 𝑏∣𝑑$$
- **展开后**：
    
    $$𝐵→𝐵𝑎𝑏∣𝑐𝑏∣𝑑$$
- **消除 B 的直接左递归**：

$$𝐵→𝑐𝑏𝐵′∣𝑑𝐵′$$
    $$𝐵′→𝑎𝑏𝐵′∣𝜀$$

核心思想是通过重排产生式，将间接左递归转为直接左递归，再消除。


## 消除回溯


回溯产生的原因是因为在读入字符时，如果按照最左推导，如果同一个非终结符两个或者多个产生式开头的字符都匹配那么则无法判断选择哪个产生式，一旦选择无法匹配就需要进行回溯

- 文法规则：`A → aB | aC`
    
- 输入符号：`a`  
    此时，分析器无法仅凭首符号 `a` 确定选择 `aB` 还是 `aC`，只能先尝试其中一个。若后续符号不匹配，则必须回溯，尝试另一候选式。


### 那么如何消除？

#### First（首字符集）

First（A），A为非终结符，代表A所有可能被推导出的第一个终结符的集合

或者通俗说First集意味着，当看到某个文法符号时，输入串的第一个字符可能是什么？

例子：

```
E → T E'  
E' → + T E' | ε  
T → F T'  
T' → * F T' | ε  
F → ( E ) | id  
```

- **计算终结符的FIRST集**：
    
    - `FIRST(+) = {+}`, `FIRST(*) = {*}`, `FIRST(() = {(}`, `FIRST()) = {)}`, `FIRST(id) = {id}`
        
- **计算非终结符的FIRST集**：
    
    - **FIRST(F)**：
        
        - 候选式 `( E )` ⇒ 首符号是 `(`
            
        - 候选式 `id` ⇒ 首符号是 `id`  
            ⇒ **FIRST(F) = { (, id }**
            
    - **FIRST(T')**：
        
        - 候选式 `* F T'` ⇒ 首符号是 `*`
            
        - 候选式 `ε` ⇒ 加入 `ε`  
            ⇒ *_FIRST(T') = { _, ε }__
            
    - **FIRST(T)**：
        
        - 候选式 `F T'` ⇒ 首符号是 `F` 的FIRST集（即 `{ (, id }`）  
            ⇒ **FIRST(T) = { (, id }**
            
    - **FIRST(E')**：
        
        - 候选式 `+ T E'` ⇒ 首符号是 `+`
            
        - 候选式 `ε` ⇒ 加入 `ε`  
            ⇒ **FIRST(E') = { +, ε }**
            
    - **FIRST(E)**：
        
        - 候选式 `T E'` ⇒ 首符号是 `T` 的FIRST集（即 `{ (, id }`）  
            ⇒ **FIRST(E) = { (, id }**



有了FIrst集那我们就能在选择产生式的时候判断该文法符号可以推导出的第一个非终结符是什么？

那么我们可以定义出消除回溯的条件为：

- **首符集（FIRST集）不相交**：  
    每个候选式的首符号集合必须互不重叠。例如，若 `A→aB | bC`，则 `FIRST(aB)={a}`，`FIRST(bC)={b}`，二者无交集。这样，看到输入符号 `a` 时，直接选择 `aB`，**无需试探其他候选式。**
    
- **无左递归**：  
    避免产生式如 `A→Aα`（如 `E→E+T`），否则分析器会无限循环。


### **如何实现消除回溯？**

1. **提取左因子**：  
    若多个候选式有共同前缀，需提取公共部分。例如：
    
    - 原产生式：`A→aB | aC`（两个候选式均以 `a` 开头）
        
    - 提取后：`A→a (B | C)`  
        这样，匹配 `a` 后，再根据后续符号选择 `B` 或 `C`。

        这里对于比较复杂的文法形式提取左因子可以有一般形式
	    ![[Pasted image 20250413214619.png]]
        
2. **计算 FIRST 集**：
    
    - `FIRST(a)` 是候选式 `a` 所有可能推导出的**首个终结符**（包括空字符 `ε`）。
        
    - 若 `A` 的候选式 `a1, a2` 满足 `FIRST(a1) ∩ FIRST(a2) = ∅`，则根据输入符号可唯一确定候选式。




### $\epsilon$ 所带来的问题


**非终结符 A** 需要匹配当前输入符号 **a**，但 **a** 不在 A 的任何候选式的“首符集”（即 A 的所有可能开头的符号）中。
然而，A 的某个候选式可以推导为**空（ε）**（即什么都不匹配）。

**关键问题**：  
这时候能否让 A 自动“跳过”（匹配空），从而继续分析后续符号？


举例：
$S\to AB$

$A\to c | \epsilon$

$B\to b$

对于这个文法来说我们现在要匹配a，那么对于非终结符A来说，A的产生式是没有实际符合条件的终结符，但是可以推导为  $\epsilon$,  但是这就导致了再继续推导B的时候会发生语法错误因为此时B只能推导出b而不是a

那么这个时候语法就一定要报错
而如果此时
$B\to a$

那么这个时候对于A来说匹配  $\epsilon$，就是没有问题的



也就是

> **输入符号 a** 必须是文法允许**紧跟在 A 后面**的符号。

否则，即使 A 可以匹配空，输入符号 a 在此处出现也是非法的（语法错误）。


### Follow集的定义

**Follow集**是编译原理中的一个概念，表示在文法中某个**非终结符A**后面可能直接出现的**终结符集合**


 例如上面例子中的Follow（A）其实就是First(B)

### **Follow集的定义与计算规则**

1. **基本定义**：
    
    - `Follow(A)` 是所有可能**紧跟在非终结符A后面**的终结符集合。
        
    - 若A是文法的开始符号，则`Follow(A)`初始包含结束符`$`。
        
2. **计算规则**：
    
    - **规则1（直接跟随）**：  
        若存在产生式 `B → α A β`，则 `Follow(A)` 包含 `First(β)` 中所有非ε的终结符。
        
        - **例**：`B → A C`，若 `First(C) = {c, d}`，则 `Follow(A) = {c, d}`。
            
    - **规则2（β推导为空时）**：  
        若 `β` 可以推导为ε（即 `β ⇒* ε`），则 `Follow(A)` 还需包含 `Follow(B)`。
        
        - **例**：`B → A C`，若 `C ⇒* ε`，则 `Follow(A) += Follow(B)`。
            
    - **规则3（末尾位置）**：  
        若存在产生式 `B → α A`（A在产生式末尾），则 `Follow(A)` 包含 `Follow(B)`。
        
        - **例**：`B → A`，则 `Follow(A) += Follow(B)`。
            
3. **迭代计算**：
    
    - 重复应用上述规则，直到所有Follow集不再变化。



```
E → T E'  
E' → + T E' | ε  
T → F T'  
T' → * F T' | ε  
F → ( E ) | id  
```

**步骤**：

1. **初始设置**：
    
    - `Follow(E) = {$}`（E是开始符号）。
        
2. **根据产生式逐条分析**：
    
    - **E → T E'**：
        
        - T后面是E'，因此 `Follow(T) += First(E')`。
            
        - `First(E') = {+, ε}`，所以 `Follow(T) += {+}`。
            
        - 又因E'可能为空，所以 `Follow(T) += Follow(E) = {$}`。  
            ⇒ **Follow(T) = {+, $}**
            
    - **E' → + T E'**：
        
        - T后面是E'，因此 `Follow(T) += First(E') = {+}`（已包含）。
            
        - E'可能为空，所以 `Follow(T) += Follow(E') = Follow(E) = {$}`（已包含）。
            
    - **T → F T'**：
        
        - F后面是T'，因此 `Follow(F) += First(T')`。
            
        - `First(T') = {*, ε}`，所以 `Follow(F) += {*}`。
            
        - T'可能为空，所以 `Follow(F) += Follow(T) = {+, $}`。  
            ⇒ **Follow(F) = { * , +, $}**
            
    - **F → ( E )**：
        
        - E后面是`)`，因此 `Follow(E) += {)}`。  
            ⇒ **Follow(E) = {$, )}**

### LL（1）文法分析的实现

根据LL（1）文法的条件和分析规则我们就可以对输入的串进行语法分析，现在讨论如何实现具体的分析步骤

**预测分析表**

预测分析表是一个矩阵，$M[A,a]$

![[Pasted image 20250414113637.png]]

![[Pasted image 20250414113631.png]]

![[Pasted image 20250414122639.png]]

如图所示这是一个预测分析表和一个文法对应，行表示非终结符，列代表终结符，终结符为文法当中的终结符而且包含一个串的终止符

表的构造规则：

如果该文法是LL（1）文法
对每个产生式 **A → α**：

1. **处理FIRST(α)**：
    
    - 对每个终结符 **a ∈ FIRST(α)**，将 **A → α** 填入表项 **M[A, a]**。
        
    - **例外**：若 **ε ∈ FIRST(α)**，还需处理FOLLOW集。
        
2. *处理空候选式（若α ⇒_ ε）**：
    
    - 对每个终结符 **b ∈ FOLLOW(A)**，将 **A → α** 填入表项 **M[A, b]**。
最后空的项都代表出现语法错误

通俗解释，构造表只需利用FIRST集和FoLLow集和即可，如果某个非终结符的产生式的First集中有终结符，那么就把对应的产生式填在  M[A, a]**中，如果这个非终结符可以推导为空，那么就去找FoLLOw集中的终结符，并把推导为空的产生式填入对应的**M[A, b]中


#### 程序如何实现LL（1）文法分析

这里还需要利用一个栈结构，这个栈用来存放所有的文法分析符号

#### **预测分析程序工作流程**

1. **初始化**：
    
    - 栈初始化为 `[#, 开始符号]`（如 `[#, E]`）。
        
    - 输入指针指向第一个符号。
        
2. **循环处理**：  
    重复以下步骤，直到栈顶和输入均为 `#`（成功）或报错（失败）：
    
    - **Step 1**: 查看栈顶符号 `X` 和当前输入符号 `a`。
        
    - **Step 2**: 根据 `X` 的类型决定操作：
        
        - **Case 1**: `X` 是**终结符**：
            
            - 若 `X == a`，弹出 `X`，输入指针后移。
                
            - 若 `X ≠ a`，报错（输入不匹配）。
                
        - **Case 2**: `X` 是**非终结符**：
            
            - 查表 `M[X, a]`：
                
                - 若表项为产生式 `X → α`，弹出 `X`，将 `α` **逆序压栈**（如 `α = Y Z`，则压入 `Z`、`Y`）。**这里之所以要采用逆序压栈是因为这里我们采用的最左推导，要保证栈顶的元素是最左边的元素所以需要进行逆序压栈**
                    
                - 若表项为空，报错（非法符号组合）。
                    
        - **Case 3**: `X` 是**结束符 `#`**：
            
            - 若输入符号也为 `#`，分析成功结束。
                
            - 否则报错（多余符号）。

栈处理的例子：
**输入串**：`id + id * id #`     #为字符串结尾的终止符

**分析过程**（逐步演示）：

| 栈               | 剩余输入             | 动作                                           |
| --------------- | ---------------- | -------------------------------------------- |
| `[#, E]`        | `id + id * id #` | 查 `M[E, id] = E→T E'`，弹出 `E`，压入 `E' T`       |
| `[#, E' T]`     | `id + id * id #` | 查 `M[T, id] = T→F T'`，弹出 `T`，压入 `T' F`       |
| `[#, E' T' F]`  | `id + id * id #` | 查 `M[F, id] = F→id`，弹出 `F`，压入 `id`           |
| `[#, E' T' id]` | `id + id * id #` | 匹配 `id`，弹出 `id`，输入指针后移                       |
| `[#, E' T']`    | `+ id * id #`    | 查 `M[T', +] = T'→ε`，弹出 `T'`，压入 `ε`（实际不压栈）    |
| `[#, E']`       | `+ id * id #`    | 查 `M[E', +] = E'→+ T E'`，弹出 `E'`，压入 `E' T +` |
| `[#, E' T +]`   | `+ id * id #`    | 匹配 `+`，弹出 `+`，输入指针后移                         |
| `[#, E' T]`     | `id * id #`      | 继续展开 `T`...（后续步骤省略）                          |

最终栈和输入均变为 `#`，分析成功。



依靠预测分析表和栈结构，就能用高级语言实现文法分析了，**这里需要注意的是这里的预测表可以对任何文法结构进行预测分析，对于有些文法结构来说，可能会有左递归和回溯情况，那么分析表里可能会有多个产生式，而对于LL（1）文法来说，表中的产生式是唯一的**


#### 错误处理

在预测分析过程中，当遇到语法错误时，需要通过合理的错误恢复策略继续分析。

 **1. 语法错误的两种情形**

1. **栈顶终结符与输入符号不匹配**：  
    例如，栈顶为 `+`，而输入符号为 `id`，无法直接匹配。
    
2. **非终结符A在栈顶，但分析表项`M[A,a]`为空**：  
    例如，非终结符 `E` 面对输入符号 `*`，但 `M[E, *]` 无对应产生式。



 **2. 错误恢复的核心思想**

跳过输入符号直到遇到“同步符号”（Synchronizing Token），使分析能够继续。同步符号集的选择直接影响恢复效果。

#### **3. 同步符号集的选择策略**

##### **策略1：使用FOLLOW(A)作为同步符号集**

- **操作**：当非终结符A在栈顶且出错时，跳过输入符号直到遇到 `FOLLOW(A)` 中的符号，弹出A，继续分析。
    
- **例子**：若 `A` 是表达式，其 `FOLLOW(A)` 包含 `;` 和 `)`，则跳过输入直到读入的字符出现 `;` 或 `)`。
    
- **局限性**：某些关键符号可能不在 `FOLLOW(A)` 中。例如，C语言中语句后的分号缺失可能导致下一语句的关键字被跳过。
    

##### **策略2：补充FIRST(A)到同步符号集**

- **操作**：将 `FIRST(A)` 中的符号加入同步集，当输入中出现 `FIRST(A)` 中的符号时，尝试恢复。
    
- **例子**：若 `A` 是语句，`FIRST(A)` 包含 `if`、`while`，则遇到这些符号时可重新开始分析 `A`。

	
##### **策略3：空产生式的默认处理**

- **操作**：若 `A` 可推导为空（`A → ε`），在错误时默认选择空产生式，推迟错误检查。
    
- **优点**：减少恢复时需要处理的状态。
    
- **例子**：`A → α | ε`，若 `α` 匹配失败，尝试使用 `ε` 继续。


##### **策略4：直接处理终结符不匹配**

- **操作**：若栈顶是终结符 `X`，但 `X ≠ a`，弹出 `X`，报错“插入X”，继续分析。
    
- **例子**：栈顶为 `+`，输入为 `id`，弹出 `+`，报错”缺失+”，继续处理 `id`。
