## 线程优先级

每个线程都有优先级，优先级决定了线程调度的优先情况，Java中的线程优先级最高为10，优先级越高说明虚拟机越优先调度这个线程执行用 setPriority来设置优先情况


### 守护线程

可以将一个线程设置为守护线程，当存在多个线程时，守护线程也执行，而当其他线程都执行完之后守护线程也会随之结束，不管这个时候有没有执行完

用setDaemon来设置


## 礼让和插队

## 线程安全

当线程执行过程中，CPU的使用权可能随时在变化，也就是说多个线程在执行中的过程是不确定的，如果多个线程减需要共享某些数据，有可能上个线程刚修改完这个数据下个线程就直接使用这个数据了，这样会造成错误，也就是操作系统中竞争资源的问题


Java中的解决方法是使用**同步代码块**

![[Pasted image 20250508135152.png]]

同步代码块会将当中的代码当成原子操作进行，确保执行的独立性，这样其他线程就无法在其中修改数据

![[Pasted image 20250508135431.png]]

当中的锁对象参数必须要**唯一**，可以是任意对象


![[Pasted image 20250508135720.png]]
这里的锁对象可以用当前类的字节码文件来替代，保证唯一性

同时需要注意的是，锁最好设置在循环内部，否则会导致需要整个循环执行完毕，才会允许其他线程执行，这会导致低效率


### 同步方法

可以将锁运用于整个方法，那这个方法就是同步方法
![[Pasted image 20250508140209.png]]

需要注意的是同步方法的锁对象无法自己指定，静态和非静态的方法不同

除了使用synchronized外还可以使用lock锁，lock锁需要手动加锁和释放
![[Pasted image 20250508141233.png]]

**这里同时也需要保证锁对象的唯一性**

之所以锁要唯一是因为锁本身也是两个线程的共享资源，只有唯一才能发挥作用

![[Pasted image 20250508141336.png]]

释放锁的时候可能释放锁这条代码写在末尾有些时候不会被执行，这就会导致其他线程一直处于等待状态

解决方法是利用try catch中的finally，而finally中的代码一定会被执行，将释放锁写在其中即可


## 生产者消费者问题



生产者产生资源，消费者消费资源

![[Pasted image 20250508163530.png]]

对于生产者和消费者来说可以看做两个线程，需要解决的问题就是如何让这两个线程都能执行且互不冲突，以上就是等待唤醒的机制


![[Pasted image 20250508163426.png]]
需要用到这几个方法来实现，这里唤醒和等待的方法具体要唤醒谁和谁等待和锁对象有关，调用方式是lock.wait()表示当前线程和这个锁有关陷入等待

![[Pasted image 20250508170443.png]]


![[Pasted image 20250508170516.png]]

如图所示，wait()和notify（）都和锁有关


在 Java 中，`wait()` 和 `notify()` 方法虽然没有参数，但它们的线程等待和唤醒机制是通过 **对象监视器锁（Object Monitor）** 隐式关联的。具体来说，线程的等待和唤醒是基于 **同一个对象实例** 的锁机制实现的。以下是关键原理和使用方法：
### 核心机制

1. **对象监视器锁**
    
    - `wait()` 和 `notify()` 是 `Object` 类的方法，必须在 **同步代码块（`synchronized` 块）** 中使用。
        
    - 线程调用 `obj.wait()` 时，会释放 `obj` 的锁并进入等待状态。
        
    - 其他线程调用 `obj.notify()` 或 `obj.notifyAll()` 时，会唤醒在该 `obj` 上等待的线程。
        
2. **隐式绑定**
    
    - 所有等待和唤醒操作都绑定到 **调用方法的对象**（即 `obj`）。
        
    - 线程的等待和唤醒通过 **共享同一个对象** 隐式关联，无需显式指定目标线程。




### 阻塞队列

![[Pasted image 20250508173529.png]]

![[Pasted image 20250508173544.png]]


阻塞队列可以看成是线程之间共享的资源，只不过这个队列在底层已经实现了相应的锁和唤醒等待机制，线程之间可以直接往里放资源和拿取资源而不必担心导致错误

![[Pasted image 20250508194043.png]]

![[Pasted image 20250508194301.png]]


这里通过submit向线程池提交任务，提交之后线程池会创建对应的线程开始执行直到执行完毕
submit接收的参数是一个继承了Runnable接口的对象或者使用Callable接口和Future接口也可以
