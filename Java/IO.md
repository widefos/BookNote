## FileInputStream（字节输入流）



字节输入流用来从文件中读入数据
![[Pasted image 20250429171258.png]]

调用read代表只读取一个字符

![[Pasted image 20250429172234.png]]

read()可以按数组大小读取，用数组读取的时候返回的每次读取的字节长度

**这里需要注意**，read()按数组读取的时候会覆盖原先的数组，但如果读取长度不够就不会完全覆盖之前的数组，造成错误

解决方法是按如下方式读取：


![[Pasted image 20250429173116.png]]

String（bytes,0,len）用来将bytes数组转换为字符串,0代表数组起始索引，len代表转换长度

### 原理

字符输入流有优先从缓冲区中读取数据，当缓冲区中没有数据再从文件中读入缓冲区
![[Pasted image 20250503175913.png]]


字符输入流会从文件中读取字符的时候会先将字符读入缓冲区中，每次调用read()函数，就会先看看缓冲区中有没有数据

## 序列化和反序列化


我们需要将对象非明文保存到文件中可以使用序列化


![[Pasted image 20250504153421.png]]


这样可以将对象保存到文件中去
**这里需要注意，序列化对象的构造函数参数是一个基本流对象**
![[Pasted image 20250504153539.png]]


**需要注意**，如果需要序列化一个对象，这个对象要继承对应的序列化接口，表示这个对象是可以序列化的，不然会报错

![[Pasted image 20250504153720.png]]


### 反序列化


![[Pasted image 20250504154210.png]]


当对象保存之后，原代码中对象的内容发生了改变，这个时候要反序列化读取对象就会报错


解决方法就是给这个对象一个固定的序列化版本UID，这个值可以在IDEA中自动生成
![[Pasted image 20250504154825.png]]



**如果需要序列化多个对象，可以先对象放入集合中然后序列化这个集合就行**
