在 Java 中，**函数式接口（Functional Interface）** 是一个**只有一个抽象方法**的接口。它是 Lambda 表达式和方法引用的基础，允许开发者以更简洁的方式实现接口的单一功能。

```
// 正确：一个抽象方法
@FunctionalInterface
interface MyInterface {
    void doSomething();
}
```

函数式接口当中只允许有一个抽象方法，且不能少，可以有其他方法但不能是抽象方法


Java 内置了多个函数式接口（位于 `java.util.function` 包）

## Comparator


`Comparator` 是一个**函数式接口**，用于定义对象的**自定义排序规则**。它通常用于对集合（如 `List`、`Set`）中的元素进行排序，尤其是在无法修改对象本身的类定义（例如无法实现 `Comparable` 接口）时，或者需要动态指定多种排序规则时。


当中有一个核心方法

```
int compare(T o1, T o2);
```

- 返回值的含义：
    
    - **负数**：`o1` 应排在 `o2` 前面。
        
    - **零**：`o1` 和 `o2` 视为相等（排序时不改变顺序）。
        
    - **正数**：`o1` 应排在 `o2` 后面。


一般如果要使得从小到大排序那可以返回o1-o2，反之


## Comparable接口


在 Java 中，**`Comparable` 接口**（而非 `CompareTo` 接口）是用于定义对象的**自然排序规则**的核心接口。它通过 `compareTo` 方法实现对象之间的比较，使对象自身具备排序能力。与 `Comparator`（外部比较器）不同，`Comparable` 是对象**内部实现**的排序规则。

需要实现排序的对象继承这个接口

并实现其中的CompareTo方法


```
class Person implements Comparable<Person> {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public int compareTo(Person other) {
        return this.age - other.age; // 按年龄升序排序
    }

    // Getter 方法省略
}
```



## lamda表达式


lamda表达式是对匿名内部类的一种简化形式，它只能用来表示函数式接口的继承**对象**
